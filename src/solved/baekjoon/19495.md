# 19495. [Square Function](./19495.cpp)

내 백준 마지막 루비 문제다. 일단 출처와 문제 길이부터 범상치 않다.

어차피 이 문제 해설을 읽으러 온 사람이면 문제 이해는 됐을거라고 생각한다. 이걸 읽을 사람이 있을지부터 의문이긴 한데.

이 문제에서 정의하는 $S(x)$는 크누스의 Concrete Mathematics 의 Chapter 4 연습문제에서 등장하며, 해당 연습문제는 이 함수가 일대일 대응인 것을 증명하는 것이다.

이를 몰라도 문제를 푸는데 지장은 없지만 알면 매우 쉬워진다. 이 글도 일대일 대응을 밑바탕에 깔고 작성할 것이다.

아무튼, 역변환을 위해서는 정변환이 어떻게 이루어지는지부터 파악해야 한다.

수를 여럿 골라서 제곱수를 만들어야 하므로 다음과 같이 수를 소수의 지수 형태로 변환한다.

2 = [1, 0, 0, ...]
3 = [0, 1, 0, ...]
4 = [2, 0, 0, ...]
5 = [0, 0, 1, ...]
6 = [1, 1, 0, ...]

여기서 더 나아가서, 제곱수면 지수가 짝수면 되니까 전부 2로 나눈 나머지로 바꿔버린다.

2 = [1, 0, 0, ...]
3 = [0, 1, 0, ...]
4 = [0, 0, 0, ...]
5 = [0, 0, 1, ...]
6 = [1, 1, 0, ...]

우리의 목표는 이제 이 배열들을 잘 골라서 모든 항이 0이 되게끔 할 수 있는 최소한의 범위를 찾는 것이다.

2를 예로 들면, (2, 3, 6)을 골라 각 열 별로 더하면 전부 2의 배수가 되어 제곱수가 되는 것을 알 수 있다.

이걸 좀 고급진 언어로 표현하자면 $\mathrm{GF}(2)$에서의 선형 결합 결과가 전부 $0$이면 되고, 따라서 $S(X) = Y$를 찾기 위해서는 $X \sim Y$까지의 범위의 수들을 전부 벡터로 환원해서 기저 벡터를 찾은 뒤, $\operatorname{rank}$의 크기가 모든 원소 수($Y - X + 1$)보다 작거나 같은 최소의 $Y$를 찾으면 된다.

말은 거창한데 사실 $X$부터 $Y$까지 전부 지수 패리티 벡터로 변환하고, 그걸 행으로 하는 행렬에서 가우스 소거 돌린 다음에 전부 0인 행이 남는지 확인하면 된다. 선대에서 흔히 말하는 선형 종속 어쩌고 하는 그 내용이다.

그래서 행을 하나씩 추가하면서 가우스 소거를 돌리다가, 영행이 나타나면 그게 함수값이 된다.

역함수도 똑같은 맥락인게, 이 함수가 일대일 대응이라서 역상이 유일하게 존재함이 보장되므로 $Y$부터 1씩 줄이면서 행을 하나씩 추가하고, 가우스 소거를 돌려서 영행이 등장하는 $X$가 역함수값이 된다.

대충 여기까지 이론적인 배경을 깔고, 어떻게 2초안에 계산을 끝낼지 알아보자. 행을 하나 추가할때마다 가우스 소거를 돌리는건 말이 안된다.

그래서 행을 하나씩 추가할때마다 기저를 빠르게 갱신할 수 있는 자료구조가 필요한데, 영어권에서는 Xor Basis라고 부르는 모양이다. 자세한 내용은 [여기](https://usaco.guide/adv/xor-basis?lang=cpp)를 참고하자. 내가 쓰는 글보다 100배쯤 영양가가 있을 것이다.

Xor Basis에 대한 구현은 [여기](./19495.cpp#65)이다.

해당 자료 구조의 행 삽입 시간 복잡도는 러프하게 따져서 $O(N^2)$이다. 기본 가우스 소거는 $O(N^3)$이니, 지수 하나가 줄어든 셈이다. 원래는 차원 어쩌고 하는걸로 따져야 하는데 이게 논문도 아니고 알빠노다.

이 문제 기준으로 $Y$의 최대값은 $10^6$이고, 이 이하의 소수는 대충 만개쯤 되는데, 이 크기가 $N$에 고스란히 박히면 Xor Basis고 나발이고 못 버틴다. 그래서 큰 소수를 따로 생각하는 방법을 택해야 한다.

$10^3$을 초과하는 소수는 $10^6$이하의 수의 소인수분해에서 하나밖에 존재하지 않으므로 $10^3$ 보다 작은 $168$개의 소수는 비트셋으로, 큰 소수는 개별적으로 하나씩 모아서 관리해주면 $10^5 - 168$ 개의 행을 날려버릴 수 있고, $N$을 $168$로 줄여버릴 수 있다.

그래서 [이런 구조체](./19495.cpp#142)를 작성했다. 168개의 소수를 bitset으로 표현하고, 큰 소수 하나를 멤버로 갖는다. 왜 크기를 $2^8$로 잡았냐면 2의 지수꼴이 아닌 수는 예쁘지 않기 때문이다.

그런 다음, $Y$부터 1씩 줄여가면서 PrimeProfile 구조체를 만들어 Xor Basis에 넣고 큰 소수가 있다면 언제 등록됐는지 여부도 같이 알려준다.

그러면서 만약 XorBasis에서 선형 종속이 발생하고 + 타임스탬프가 $Y$라면 역함수값으로 인정하고 결과에 추가한다. 타임스탬프가 $Y$와 다르다면, 즉 $Y$보다 작다면 그 값의 함수값이 $Y$가 아니기 때문이다.

한 가지 첨언하자면 CM에서는 모든 합성수가 $S$의 치역이라고 언급했다. 따라서 소수면 -1 출력, 아니면 일대일 대응이므로 찾아서 바로 출력하는 식으로 해도 될 것이다.

아 글쓰기 힘들다. 쇼츠는 안 보지만 그래도 팝콘 브레인의 소유자가 된 것 같다.
