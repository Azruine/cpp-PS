# 2657. [Find the Prefix Common Array of Two Arrays](./2657.cpp)

배열 최대 크기가 50이고, $1$부터 $N$ 까지의 원소가 한 번씩 등장하는게 보장되므로 64비트 정수에다 비트 켜준 다음 같이 켜진 개수를 구하면 된다.

사실 이게 더 C++ 스러운 코드라고 생각한다.

```cpp
#include <vector>

class Solution {
public:
    std::vector<int32_t> findThePrefixCommonArray(std::vector<int32_t>& A,
                                                  std::vector<int32_t>& B) {
        auto n = A.size();
        std::vector<int32_t> ret(n, 0);
        uint64_t a_bit = 0, b_bit = 0;
        for (auto i : _iota(n)) {
            a_bit |= 1UL << A[i];
            b_bit |= 1UL << B[i];
            ret[i] = std::popcount(a_bit & b_bit);
        }
        return ret;
    }
};
```

체이닝을 러스트에서 짜면 이렇게 나온다.

```rust
#[allow(unused_imports)]
use itertools::Itertools;

impl Solution {
    pub fn find_the_prefix_common_array(a: Vec<i32>, b: Vec<i32>) -> Vec<i32> {
        a.into_iter()
            .zip(b)
            .scan((0u64, 0u64), |state, (x, y)| {
                state.0 |= 1u64 << x;
                state.1 |= 1u64 << y;
                Some((state.0 & state.1).count_ones() as i32)
            })
            .collect()
    }
}
```

CPP에 아직 views::scan이 없기 때문에, transform 에 붙는 unary operator를 mutable로 선언해서 state를 수정하면서 scan을 흉내내야 한다. 왜 accumulate랑 inclusive_scan을 view로 안 만드는지 모를 일이다. scan은 심지어 [제안서](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2025/p3351r2.html)까지 올라가있다.

그래서 transform 에 mutable 달아서 가는거랑 러스트의 scan이랑 무슨 차이냐 하면, C++에서 view는 여러 번 호출될 수 있다. Rust에서는 iterator는 끝에 도달해서 None을 반환하면 수명이 아예 종료되는 것과는 대조적이다.

그런데 mutable로 들어있는 변수는 view 자체에 속한거라, view를 여러번 순회하면 mutable 변수, 그러니까 state가 누적이 되어버린다.

마치 static 변수를 대충 선언했을때 만나는 상황과 비슷하다.

```cpp
std::vector arr = {0, 1, 2, 3};

auto view = arr | std::views::transform([sum = 0](auto x) mutable {
                return sum = sum + x;
            });

for (auto elem : view) {
    std::cout << elem << ' ';
}
for (auto elem : view) {
    std::cout << elem << ' ';
}
/*
0 1 3 6 6 7 9 12
*/
```
