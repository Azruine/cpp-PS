# 1674. [Minimum Moves to Make Array Complementary](./1674.cpp)

배열을 접어서 그 합이 일정하게 만들어야 한다.

숫자를 $[1, limit]$ 내에서 한 번 변동시킬 수 있으니, 하나의 쌍에 대해 $0, 1, 2$ 번 바꿀 수 있다.

뒤집어서 생각하면 $i$ 번째 인덱스 쌍에서 다음 세 가지 경우가 가능하다.

1. 숫자를 0번 바꾸면 합은 $nums[i] + nums[n - i - 1]$ 가 된다.
2. 숫자를 1번 바꾸면 합을 $[min(nums[i], nums[n - i - 1]) + 1, max(nums[i], nums[n - i - 1]) + limit]$ 구간 안에서 조정할 수 있다.
3. 숫자를 2번 바꾸면 합을 $[2, limit * 2]$ 구간 안에서 조정할 수 있다.

따라서 $[2, limit * 2]$ 에 대해서 각 쌍 별로 숫자를 몇 개 조작하면 해당 구간에 안착할 수 있는지 표기한다.

```cpp
for (auto i = 0UL; i < (n >> 1); i++) {
    auto sum = nums[i] + nums[n - i - 1];
    auto lower = std::min(nums[i], nums[n - i - 1]) + 1;
    auto upper = std::max(nums[i], nums[n - i - 1]) + limit;
    diff[sum] += 1;
    diff[sum + 1] -= 1;
    diff[lower] += 1;
    diff[upper + 1] -= 1;
}
```

다음, 차분 배열을 쭉 순회하면서 prefix sum 배열을 만든다. 이 prefix 배열은 쌍의 합을 $i$로 만들 때, 변경이 """안""" 되는 숫자의 갯수가 된다.

따라서 prefix 배열 전체에서 최대값을 찾고, 그걸 $nums$ 배열의 크기에서 뺀 값을 반환하면 맞는다.
